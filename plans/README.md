## Direction findings

- Add an optional summary CSV for repeated seeds after Plan 001. Raw per-seed output is useful for auditability, but an averaged `results/qos_summary.csv` would make report-writing easier. Trade-off: extra file and extra validation surface.
- Consider documenting whether throughput is "delivered bytes over offered traffic duration" or "delivered bytes over simulation duration." The current design fixes `measurementDuration = trafficStop - trafficStart`, which is defensible, but the large-buffer drain period makes interpretation important in reports.
